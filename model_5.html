Our Implementation

First, a parameter grid is defined with various values for hyperparameters, including the number of estimators (trees) in the ensemble, learning rate, maximum depth of trees, and subsampling rate. Next, an XGBoost Classifier ('xgb_clf') is initialized with a specified random state. GridSearchCV is then utilized to perform a hyperparameter search over the defined parameter grid. The GridSearchCV object ('grid_search_xgb') is configured with 5-fold cross-validation and the F1 score as the evaluation metric, appropriate for imbalanced datasets. By fitting the GridSearchCV object to the training data ('X_train_ds' and 'y_train_ds'), the optimal combination of hyperparameters is determined, and the best parameters (best_params_xgb) are obtained.

Using the identified best parameters, a new XGBoost model ('xgb_best') is created and trained on the balanced training dataset. The resulting model is then utilized to make predictions on the test set ('X_test'), and the predicted results are stored in 'y_pred_up'. Additionally, the classification report is generated and printed, providing a detailed overview of the model's performance on the balanced data, including metrics such as precision, recall, and F1 score.
